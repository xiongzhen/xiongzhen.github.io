Graph Element Reference to Column Data

Posted on 2019-06-12 |

That being capable of customizing graph elements with reference to existing data is among the most useful and most powerful features. For example, you can set the symbol size of a scatter plot by referring to numeric values, set the background color of a bar plot by referring to a categorical values. You have complete yet easy access to the graph elements.

Modified Column and Main Column

The referenced column, a.k.a modified column, must be in the worksheet containing the main column – that is, the column for the second direction of a 2D graph or for the third direction of a 3D graph.

For example, given a 2D scatter plot created from XY columns with an extra size column:

and given a 3D scatter plot created from XYZ columns with an extra size column:

Color Modifier and Internal Modifier

Color modifier is a modified column specifically used for colors. For example, the symbol color can be defined by column in one of the follow ways:

  1. Indexing
    There is a prefined color list, Black = 0, Red = 1, Green = 2 and etc. You can change the color list so it’s likely not black for 0 (the old black-red-green color list actually is no longer the system default one).
    What would happen if the modifier value is greater than the number of the colors? For example, you have 3 colors in the list, red, green and blue. A value, beyong the number of colors as 3, refers to a color back from the beginning, therefore ‘4’ would means red.
  2. Direct RGB
    A direct RGB modifier means the values of the modifier are the values of the target, in RGB. For example, if the value if 0x0000ff, it means red #0000ff.
    Wait, isn’t it green #0000ff? The COLORREF is defined as BGR instead of RGB and I suppose there must be something to do with that.
  3. Color Mapping
    This is applied to colormap. For example, the GrayScale, a palette from black to white, can be mapped to a modifier so that the smallest value refers to black and the largest value refers to white. The mapping can be modified in the “Colormap” tab in Plot Details dialog (the software is powerful in that it is totally under your control).

Internal modifier is much simpler. A symbol size modifier is an internal modifier. It means the size of symbols. That is it. I suppose color modifier used to be internal modifier as well. It is made a special one just to make it applicable to multiple scenarios.

Dataset Accessing in Origin C

Posted on 2018-11-13 |

More than mouse clicking and dragging to perform analysis and create graphs in Origin/OriginPro (simply “Origin” hereafter in this post), provided also includes the power of programming, be it LabTalk the scripting language, Origin C the ANSI C closely based programming language with C++ features, or Python the popular and convenient tool. In this post, I would described the most basic operations using Origin C: how to access dataset. It is targeted at whoever has experience in C/C++ programming and is familiar with Origin.

In this post, I’ll describe some basic concepts of dataset in Origin C and show you how to get dataset as an array in the form of vector (1-D) / matrix (2-D).

Dataset

Before starting out to write codes, it would be helpful to understand what “dataset” means in Origin. Typically, there are two kinds of datasets: one that you can see directly in a view, either a Worksheet or a Matrix Sheet, which is called associated dataset or dataset for convenience; the other is call loose dataset which is not associated with any view.

You can list all datasets by LabTalk command list s where s is short for series now known as dataset:

To refer to a particular dataset in Origin C, you can create an instance of Dataset, one of the composite data types supported, with the corresponding name:

However, Dataset is not recommended in Origin C. When programming, the input data usually is either from external source (e.g. file or data stream) or from Origin object (e.g. worksheet or matrixsheet), so is the output. A Dataset is simply a collection of numbers and does not tell much about what it belongs to. On the other hand, it does not support string array well.

Usually, I use Origin C to read some data from a worksheet/matrixsheet and save the results to a worksheet/matrixsheet. Worksheets and matrixsheets are contains of datasets.

A worksheet may have multiple columns. Each column is associated with a 1-D dataset, called vector.
A matrixsheet may have multiple matries. Each matrix is associated with a 2-D dataset, called matrix.
Data assessing in Origin C is mainly about reading and writing vectors and matrices.
